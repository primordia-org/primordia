# Project Name Brainstorm

> Goal: find names more user-friendly than **Primordia** for a self-modifying web app where users describe changes in plain English and the app rebuilds itself.

---

## The Core Concept to Name

- Users describe a change → AI writes the code → app updates itself
- No coding or git knowledge required
- The app is always becoming what its users need

---

## Candidates by Theme

### Verb / Action-oriented (approachable, active)
| Name | Rationale |
|---|---|
| **Mold** | Short, tactile — you mold the app into shape |
| **Forge** | You forge new features into existence; powerful feel |
| **Sculpt** | Creative, hands-on; implies iterative refinement |
| **Craft** | Artisanal; "crafting" software without coding |
| **Reshape** | Direct and literal; immediately communicates the core idea |
| **Rewire** | Technical-sounding but punchy; implies deep change |

### Living / Growth (warmer than Primordia, still organic)
| Name | Rationale |
|---|---|
| **Sprout** | Grows from small seeds (requests); fresh, approachable |
| **Bloom** | The app blooms into what you need over time |
| **Canvas** | Blank slate that becomes whatever you paint |
| **Clay** | Raw material you shape; tactile and simple |
| **Loom** | Weaves features together; slightly poetic |

### Self-modification / Meta
| Name | Rationale |
|---|---|
| **Morph** | Obvious, direct, fun — immediately understood |
| **Flux** | Always changing, fluid; has a tech-cool feel |
| **Shift** | Lightweight; each request shifts the app |
| **Reflex** | The app reflexively responds to what you need |
| **Echo** | Your words echo back as real features |

### Conversational / AI-forward
| Name | Rationale |
|---|---|
| **Wishcraft** | Whimsical — "make a wish and it's built"; memorable |
| **Wordcraft** | Your words become code; ties language to creation |
| **Plainly** | Emphasises plain-English input; friendly, no-jargon |
| **Spellwork** | Words "cast" changes; magical metaphor for non-coders |
| **Utterance** | You utter a change and it happens; slightly formal |

---

## Top Picks (Short List)

| Name | Why It Works |
|---|---|
| **Morph** | One syllable, instantly understood, fun, no mythology baggage |
| **Clay** | Universally understood metaphor for something moldable; warm |
| **Sprout** | Friendly, growth-oriented, approachable for non-technical users |
| **Wishcraft** | Memorable, whimsical, communicates "no coding needed" |
| **Forge** | Strong, active, implies permanence — you're building something real |
| **Flux** | Modern, techy-cool, honest about the always-changing nature |

---

## Why "Primordia" Is Hard

- Primordia means "first beginnings" (primordial) — conceptually accurate but:
  - Hard to spell / remember for casual users
  - Feels sci-fi or academic, not approachable
  - Not self-explanatory — users don't know what the app does from the name alone

## Recommendation

Based on name quality + domain availability across both rounds:

| Name | Best Available Domain | Price | Notes |
|---|---|---|---|
| **Chisel** | `chisel.app` + `chisel.dev` | ~$2,490 | Best combo: great name, both domains for sale, no competing product |
| **Stitch** | `stitch.app` | Unknown | Warm, tactile; for sale; Google uses "Stitch" brand separately |
| **Morph** | `morph.app` | $239,999 | Best name but prohibitively expensive |

**Top pick: `chisel.app`** — "Chisel" perfectly captures shaping something with precision; $2,490 is a realistic price for a project domain; no active product at chisel.app or chisel.dev to conflict with.

**Runner-up: `stitch.app`** — warm, approachable, and the "stitching features together" metaphor works well. Worth checking the sale price on Dan.com.

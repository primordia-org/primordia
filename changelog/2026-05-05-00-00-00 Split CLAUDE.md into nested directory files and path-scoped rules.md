# Split CLAUDE.md into nested directory files and path-scoped rules

## What changed

The root `CLAUDE.md` was a 485-line monolith that loaded fully into Claude Code's context on every session (including all worktree sessions). The File Map alone was ~217 lines of annotated directory listings — most irrelevant for any given task.

Refactored into:

**Root `CLAUDE.md`** (140 lines) — always-loaded context:
- What Is Primordia, Tech Stack, condensed File Map with pointers to detail files
- Environment Variables, Setup Checklist
- Design Principles for Claude Code (all 10 rules)
- Current Features table, Changelog meta-note

**New subdirectory `CLAUDE.md` files** (lazy-loaded when Claude works in that area):
- `app/api/evolve/CLAUDE.md` — full evolve request data flow, session state machine (both status tables + transition table), worktree NDJSON log format
- `app/api/auth/CLAUDE.md` — RBAC model (roles, tables, bootstrap, helpers), auth-provider plugin pattern
- `lib/CLAUDE.md` — Git config key-value store pattern (namespaces, output format, code references)

**New path-scoped rules files** (`.claude/rules/filemap-*.md`) — loaded only when Claude reads files in the matching directory:
- `filemap-lib.md` (paths: `lib/**`) — full annotated listing for all lib/ files
- `filemap-app-api.md` (paths: `app/api/**`) — full annotated listing for all API route files
- `filemap-app-pages.md` (paths: `app/**`, `components/**`) — full annotated listing for pages and components
- `filemap-scripts.md` (paths: `scripts/**`) — full annotated listing for scripts/

## Why

Claude Code loads all ancestor-directory `CLAUDE.md` files at session start, but subdirectory `CLAUDE.md` files and `.claude/rules/` files with `paths:` frontmatter are lazy-loaded — only when Claude reads files in those directories. The path-scoped rules approach gives the annotated File Map a "RAG-like" quality: detail surfaces exactly when relevant, not upfront for every task.

`lib/system-prompt.ts` was confirmed orphaned (not imported anywhere) so no code changes were needed there.

# Fix install.sh sibling reparenting

## What changed

The sibling-reparenting loop in `scripts/install.sh` was using `IFS='='` when
reading output from `git config --get-regexp`.  `git config --get-regexp`
separates keys from values with a **space**, not `=`, so with `IFS='='` the
entire `key value` string was assigned to `key` and `val` was always empty.
The `if [[ "$val" == "$OLD_PROD_BRANCH" ]]` condition therefore never matched,
and no siblings were ever reparented.

## Fix

Removed the `IFS='='` override so the default `IFS` (space/tab/newline) is
used, which correctly splits `branch.foo.parent main` into
`key=branch.foo.parent` and `val=main`.

## Why it matters

Sibling reparenting is the step that updates all other in-progress sessions
(whose parent was the old production branch) to point at the newly-deployed
branch.  Without this fix, sibling sessions' "Apply Updates" button would keep
fetching diffs against the old production branch instead of the current one.

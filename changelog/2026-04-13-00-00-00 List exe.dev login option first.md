# List exe.dev login option first

## What changed

- Moved the **exe.dev** tab to the first position in the login tab switcher (was previously third, after Passkey and QR Code).
- Changed the default selected tab from `"passkey"` to `"exe-dev"` so users land on the exe.dev login option when the page loads.

## Why

The exe.dev login method should be the primary/preferred option for users of this platform, so it is now presented first and selected by default.

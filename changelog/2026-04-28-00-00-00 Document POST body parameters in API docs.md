# Document POST body parameters in API docs

## What changed

`public/openapi.json` is now git-ignored and generated via `bun run generate-api-docs` (powered by `next-openapi-gen`). The source of truth is JSDoc annotations in the route files, not the generated file.

Added `@body` JSDoc tags and exported TypeScript body-type interfaces to all POST and PATCH route handlers that accept a request body. Previously the API docs only documented query parameters and path parameters, leaving all body parameters undocumented. Also added summaries and descriptions to those handlers.

### Endpoints updated

**Evolve**
- `POST /evolve` — multipart/form-data (`request`\*, `harness`, `model`, `cavemanMode`, `cavemanIntensity`, `encryptedApiKey`, `attachments`).
- `POST /evolve/followup` — multipart/form-data (`sessionId`\*, `request`\*, `harness`, `model`, `encryptedApiKey`, `attachments`).
- `POST /evolve/manage` — JSON (`action`\* `"accept"|"reject"`, `sessionId`\*).
- `POST /evolve/abort` — JSON (`sessionId`\*).
- `POST /evolve/upstream-sync` — JSON (`sessionId`\*, `action`\* `"merge"`).
- `POST /evolve/from-branch` — JSON (`branchName`\*).
- `POST /evolve/kill-restart` — JSON (`sessionId`\*).

**Admin**
- `POST /admin/rollback` — JSON (`worktreePath`\*).
- `PATCH /admin/proxy-settings` — JSON (`previewInactivityMin`, `diskCleanupThresholdPct`).
- `POST /admin/git-mirror` — JSON (`url`\*).
- `POST /admin/permissions` — JSON (`userId`\*, `role`\* `"can_evolve"`, `action`\* `"grant"|"revoke"`).

**Auth**
- `POST /auth/cross-device/approve` — JSON (`tokenId`\*).

**LLM Key**
- `POST /llm-key/encrypted-key` — JSON (`iv`\*, `ciphertext`\*).

(\* = required field)

Each body type is an exported TypeScript interface in the relevant route file. The generator resolves it to a JSON Schema with field descriptions, required/optional markers, and enums where applicable.

### Auto-generation on first page view

Added `app/api/openapi/route.ts` — a GET handler that serves the OpenAPI spec, generating it automatically on first request if `public/openapi.json` doesn't yet exist on disk. Generation runs `bun run generate-api-docs` as a child process (the same command used manually) rather than importing `next-openapi-gen` directly — direct import causes a Next.js build error because the package uses `new URL('../../package.json', import.meta.url)` which the bundler can't resolve. Concurrent first-requests are deduplicated via a shared Promise. The written file acts as an on-disk cache for subsequent requests. Updated `app/api-docs/page.tsx` to point Scalar at `/api/openapi` instead of the static `/openapi.json`.

No build step or install step is needed. Opening `/api-docs` for the first time triggers generation automatically.

## Why

The evolve endpoints use multipart/form-data to support file attachments, which is non-obvious from the endpoint name alone. Without documented body schemas, developers using the API reference had no way to know what fields to send in POST requests.

The previous approach (manually editing `public/openapi.json`) was wrong — that file is generated by `next-openapi-gen` and should not be committed. The correct fix is to annotate the route handlers themselves so the generator produces the documentation automatically.

The `@tags` (plural) in the handler-level JSDoc blocks was a bug: `next-openapi-gen` reads `@tag` (singular), so `@tags Evolve` was parsed as tag name `"s Evolve"`. Fixed by changing all handler-level `@tags` to `@tag`. Routes without handler-level JSDoc were unaffected — the tool falls back to capitalising the first path segment as the tag name.

### Consistent summaries across all routes

Added handler-level JSDoc `/** summary \n * @tag ... */` comments directly before every exported route handler that was previously showing only its raw path in the API docs. The issue was that existing JSDoc blocks sat at the top of files (before `import` statements), where Babel does not attach them as `leadingComments` of the export declaration — so the generator never saw them. Fixed by moving those blocks to be immediately before their handler and adding new blocks for routes that had none. The deprecated `/api/rollback` and internal `/api/markdown-stream` routes are marked `@ignore` so they are omitted from the spec.

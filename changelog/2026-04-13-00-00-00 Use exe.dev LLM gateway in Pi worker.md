# Use exe.dev LLM gateway in Pi worker

## What changed

`scripts/pi-worker.ts` now unconditionally uses the exe.dev link-local LLM
gateway (`http://169.254.169.254/gateway/llm/anthropic`) instead of
`ANTHROPIC_API_KEY`.

- Sets a placeholder `"gateway"` runtime API key for the `anthropic` provider
  (the SDK requires a non-empty value; the gateway handles auth itself).
- Registers an inline extension factory via `DefaultResourceLoader`'s
  `extensionFactories` option that calls
  `pi.registerProvider('anthropic', { baseUrl: GATEWAY_BASE_URL })`, pointing
  all Anthropic requests at the gateway endpoint.

The previous probe-and-fallback logic (which fell back to `ANTHROPIC_API_KEY`
when the gateway was unreachable) has been removed.

## Why

The Pi worker runs exclusively inside exe.dev VMs where the gateway is always
available. The fallback to `ANTHROPIC_API_KEY` was masking misconfiguration and
preventing verification that the gateway codepath is actually being used.
Removing it ensures the gateway is always exercised.

# Task accordion session events

Session progress now nests the agent's text, reasoning, and tool-call event stream inside the task list UI. Each tracked task keeps the original simple task-list appearance and can be opened to reveal its associated detail events; setup-time events are grouped into a virtual "Setup" task, while the agent's final post-tool message remains below the progress section as its own message.

This keeps Pi/agent task progress and detailed execution history in one scannable accordion instead of splitting the to-do summary from the detailed log below it. The task list preserves the original Tasks header, weighted progress bar, compact/selectable task rows, compact spacing while details are closed, with breathing room before revealed tool calls, and up/down chevron expansion behavior while keeping the chevron controls at the top and bottom edges of the task list. It avoids an extra nested wrapper, redundant padding, indented disclosure content, or left-border treatment around revealed task details, uses subdued styling for boilerplate Setup rows, and marks Setup complete once real tasks are present. Task detail/tool-call reveals use the same smooth grid/opacity animation as the all-tasks expansion. Pi task tool events such as TaskCreate, TaskList, and TaskUpdate render as concise single-line entries inside the revealed details, with expected task ids omitted and status labels simplified (for example, "in progress" and "complete"); expected in-progress updates use subdued text instead of attention-grabbing yellow. Current tasks prefer their `activeForm` label when available, and Pi guidance now asks agents to batch a completion update with the next task's in-progress update in one turn.

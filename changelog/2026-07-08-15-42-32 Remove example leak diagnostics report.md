# Remove example leak diagnostics report

Removed the checked-in sample CPU/memory leak diagnostics files that were originally used to preview the Server Health diagnostics UI. Because those files shipped with every deploy, fresh deployments incorrectly showed an "example report for UI testing" under “Diagnose CPU usage / memory leaks.”

Added `leak-diagnostics/` to `.gitignore` so real runtime diagnostics can still be generated locally on affected instances without being committed as bundled example data again.

Follow-up: simplified evolve secret handling so the browser sends a signed nonce proof with an auth-source-specific X25519 public key when starting, continuing, or accepting an evolve session. Each stored credential row has its own server X25519 keypair in SQLite; the browser deterministically derives a unique WebCrypto X25519 keypair for each auth source from the local secret via PBKDF2, signs the nonce with a deterministic WebCrypto Ed25519 key, and the server derives `PRIMORDIA_DECRYPTION_KEY` only after verifying that signature. Workers receive only that derived key and read the selected encrypted credential directly from SQLite by user/auth source. Existing local secrets are migrated to versioned WebCrypto Curve25519 ciphertexts (`x25519-source-v1`) in a nondestructive way, with no manual Curve25519 implementation or fallback. A CLI-friendly `getPlaintextCredentialsForUser(userId, publicKeyOrProof, authSource)` utility remains the simple plaintext resolution path. Legacy worker credential override env vars are rejected; workers now have one credential path only: decrypt the selected SQLite row with `PRIMORDIA_DECRYPTION_KEY`. The secrets client documentation was restored around the new flow, dead compatibility helpers were trimmed, native Baseline 2025 `Uint8Array` base64url helpers are used for proof/key encoding, and unit tests now cover the WebCrypto round trip plus signed nonce proof behavior.

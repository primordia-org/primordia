# Pin Bun with mise

Primordia now includes a committed `mise.toml` that pins Bun to `1.3.13`, matching the package manager version expected by the app.

The installer now ensures `mise` is installed, configures bash activation, trusts Primordia worktree configs for non-interactive runs, runs `mise install` from the worktree so `mise.toml` controls tool versions, copies the worktree's `mise.toml` to the Primordia root for the copied reverse proxy, and writes the systemd service so it runs through mise with shims on `PATH`.

The evolve setup flow now trusts each worktree's `mise.toml` before dependency installation. The reverse proxy now starts preview and production worktree servers with `mise exec -C <worktree> -- bun ...` instead of reusing the proxy's own executable path. That lets mise resolve the Bun version from each worktree's config, so different branches can carry different runtime pins when needed.

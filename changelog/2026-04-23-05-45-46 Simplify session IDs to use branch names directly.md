# Simplify session IDs to use branch names directly

## What changed

- **Removed the `branch.{name}.sessionId` git config key.** Previously, each
  branch wrote a `sessionId` entry into git config, and the reverse proxy read
  that mapping to route `/preview/{sessionId}` requests. Since the session ID
  was already equal to the branch name, this intermediate mapping was pure noise.

- **Reverse proxy now uses branch names directly** as the key for preview
  routing. `sessionPortCache` and `sessionWorktreeCache` are now keyed by
  branch name instead of an opaque session ID.

- **Branches with slashes are silently ignored** for preview routing and session
  creation. The proxy, branches page, and from-branch API all skip or reject
  branches whose names contain `/`. This keeps URL path segments unambiguous
  and worktree directory names simple.

- **`from-branch/route.ts` simplified.** The old flow slugified the branch name
  via Claude Haiku to generate a distinct session ID (because branch names could
  contain slashes). Now slashed branch names are rejected upfront with a clear
  error, and simple branch names are used directly as the session ID — no slug
  generation, no uniqueness loop.

- **`app/branches/page.tsx` simplified.** The `BranchData.sessionId` field was
  replaced with `hasSession: boolean`. The branch name itself is used in the
  `/evolve/session/{name}` link. The fallback lookup of `branch.{name}.sessionId`
  from git config was removed. Branches with slashes in their name are filtered
  out before rendering.

- **`lib/evolve-sessions.ts`**: removed the `runGit config branch.{name}.sessionId`
  write that was executed during worktree setup.

- **`lib/session-events.ts`**: updated comments to clarify that session ID
  equals the branch name and that branches with slashes are not supported.

## Why

The session ID was introduced when branch names could contain characters (like
slashes) that are not valid as URL segments or directory names. A UUID-style
session ID was a safe neutral identifier. But the code had already simplified
to set `sessionId = branch`, making the extra indirection dead weight.
Eliminating it reduces the number of concepts (no more "what's the difference
between a session ID and a branch name?"), removes a git config write on every
session start, and makes the reverse proxy cache logic straightforward.

Ignoring slashed branches is the right trade-off: they are uncommon (most
branch names generated by Primordia are kebab-case), the error message is
clear, and the alternative (slugifying) was opaque and hard to debug.

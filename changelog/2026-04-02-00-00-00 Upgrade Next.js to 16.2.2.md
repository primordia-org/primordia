# Upgrade Next.js to 16.2.2

## What changed

- `next`: `15.5.12` → `16.2.2`
- `eslint-config-next`: `16.2.0` → `16.2.2`
- `next.config.ts`: removed the `eslint: { ignoreDuringBuilds: true }` option, which was removed from `NextConfig` in Next.js 16 (ESLint is no longer invoked during `next build`)
- `next.config.ts`: added `allowedDevOrigins: ["*.exe.xyz"]` — Next.js 16 blocks cross-origin requests to dev resources (HMR WebSocket at `/_next/webpack-hmr`) by default; without this, the dev client can't connect when accessed via the exe.dev reverse proxy, breaking React hydration and making buttons unresponsive
- `tsconfig.json`: `"jsx"` changed from `"preserve"` to `"react-jsx"` (mandatory in Next.js 16; the dev server enforces this automatically); added `.next/dev/types/**/*.ts` to `include`
- `components/EvolveSessionView.tsx`: removed the `kill-restart` call from `handleAccept` — after the worktree is accepted and deleted, calling kill-restart on the (now-deleted) worktree path caused the restart to fail and overwrote the `"accepted"` status with `"error"` in the DB; the parent server picks up merged file changes automatically via Turbopack HMR, so the restart was unnecessary
- `lib/evolve-sessions.ts`: fixed `restartDevServerInWorktree` to kill the entire process group (via negative PID) before spawning a replacement server — Turbopack spawns worker processes that don't bind to the port, so the previous `lsof`-only approach left orphaned workers that blocked the new server from binding; `lsof` kill is retained as a fallback when no in-memory process reference is available
- `lib/evolve-sessions.ts`: updated stale "Next.js 15" comment to "Next.js 16"
- `PRIMORDIA.md`: updated tech stack table to reflect Next.js 16

## Why

Next.js 16.2.2 is the latest stable release. Staying on the latest version ensures access to new App Router features, performance improvements, and security patches.

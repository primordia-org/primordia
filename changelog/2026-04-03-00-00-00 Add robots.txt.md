# Add robots.txt

## What changed

Added `public/robots.txt` to guide web crawlers.

## Why

Without a `robots.txt`, search engine bots and other crawlers would index sensitive or non-public routes (API endpoints, admin panel, evolve pipeline, login, owner shell, branches diagnostic page). This file explicitly allows crawling of the public-facing pages while blocking:

- `/api/` — backend API routes, not useful to index
- `/admin/` — admin panel (permission-gated)
- `/oops/` — owner-only command shell
- `/evolve/` — evolve pipeline UI (permission-gated)
- `/login/` — authentication pages
- `/branches/` — dev-only git diagnostics

The landing page (`/`), chat (`/chat`), and changelog (`/changelog`) remain crawlable.

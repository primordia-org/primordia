# Add Gemini 3.5 Flash to Pi

Added `gemini-3.5-flash` as a supported Pi/Gemini model before the bundled pi model registry exposes it directly.

What changed:

- Added a Primordia-owned pi `models.json` overlay containing the `gemini-3.5-flash` Google Generative AI model definition.
- Wired the Pi worker and evolve slug generator to create their `ModelRegistry` instances with that overlay, so sessions can resolve and run the model instead of falling back to defaults.
- Added Gemini 3.5 Flash to the Evolve model picker and updated the built-in Gemini preset to select it.
- Updated the model-registry regeneration script so future regenerations keep Primordia's overlay models and direct Google provider entries.

Why:

Google Gemini 3.5 Flash support is needed in Primordia's Pi harness before the upstream `@mariozechner/pi-coding-agent` package publishes the model in its built-in registry.

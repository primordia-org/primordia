# Add Gemini 3.5 Flash to Pi

Added `gemini-3.5-flash` as a supported Pi/Gemini model, including both direct Google API key usage and OpenRouter usage, before the bundled pi model registry exposes it directly.

What changed:

- Added a Primordia-owned pi `models.json` overlay containing the direct `gemini-3.5-flash` Google Generative AI model definition and the `google/gemini-3.5-flash` OpenRouter model definition.
- Wired the Pi worker and evolve slug generator to create their `ModelRegistry` instances with that overlay, so sessions can resolve and run the model instead of falling back to defaults.
- Added Gemini 3.5 Flash to the Evolve model picker for both Google Gemini API keys and OpenRouter API keys.
- Updated the built-in Gemini preset to select direct Gemini 3.5 Flash, and added a built-in OpenRouter + Gemini 3.5 Flash preset.
- Updated the model-registry regeneration script so future regenerations keep Primordia's direct Google and OpenRouter overlay model entries.

Why:

Google Gemini 3.5 Flash support is needed in Primordia's Pi harness before the upstream `@mariozechner/pi-coding-agent` package publishes the model in its built-in registry. OpenRouter support lets users run the same model with their existing OpenRouter billing source instead of requiring a separate Google Gemini API key.
